# ADR-0001: TypeScript with ES Modules

**Status:** Accepted  
**Date:** 2026-07-01

## Context

We need to choose a language and module system for Codexia.

## Decision

Use TypeScript with ES Modules (ESM).

- TypeScript for type safety and better tooling
- ESM for modern Node.js compatibility
- Target ES2022 for modern features

## Consequences

- Requires `.js` extensions in imports for ESM compatibility
- Need to configure `tsconfig.json` for NodeNext module resolution
- Better IDE support and refactoring capabilities
- Slightly more complex build setup
